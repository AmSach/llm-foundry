LLM Foundry

A modular system for useful long-horizon AI

LLM Foundry is not just a model. It is the workshop around a model. It gives you memory, compression, tools, traces, training, and tests so the system can do real work instead of just sounding smart.

Model agnostic Memory aware Tool ready Evaluation first

Explain it like a kid

Think of the model as the brain, the tokenizer as the translator, memory as the notebook, compression as the editor, tools as the hands, and harnesses as the quizzes.

That is how the system keeps important facts, uses fewer tokens, and gets better over time.

What it is not

It is not a frontier-trained base model yet. It is the control plane around a model, which is what you need if you want long-term useful behavior, memory, and agent work.

The project is designed to plug into existing models or future custom models without rewriting the rest of the stack.

Why it helps with long-term decision making

  1. It compresses long context into a compact prompt.
  2. It saves important facts in an Obsidian-style memory vault.
  3. It lets the model call tools instead of guessing.
  4. It records traces, so good behavior can become training data.
  5. It runs harnesses, so you can measure whether the system really improved.

Use cases

  • long-running assistants
  • research and analysis agents
  • coding helpers
  • document and knowledge workflows
  • model experimentation and training

Advantages over plain chat models

  • less wasted context
  • better recall of important facts
  • tool use for real tasks
  • trace export for learning
  • repeatable testing

Integration

  • OpenAI-compatible backends
  • Hugging Face backends
  • custom adapters that expose generate(prompt)
  • deterministic test backends

How to use it

python -m llm_foundry smoke-test python -m llm_foundry benchmark --backend echo --output benchmark.json --markdown benchmark.md python -m llm_foundry compress --task "Summarize this" --transcript-file path/to/transcript.txt python -m llm_foundry harness --backend echo python -m llm_foundry train-model --corpus examples/corpus.txt --config model.json

Who should use it

If you want a system that can remember, compress, check, and act, this is the right layer. If you only want a chatbot, it is probably too much.

If you want a model that can power product workflows, long tasks, and agent loops, this is the part that makes that possible.
